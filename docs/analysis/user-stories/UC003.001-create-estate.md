# UC003.001 — Create Estate

## Overview

| Attribute | Value |
|---|---|
| **ID** | UC003.001 |
| **UC** | [UC004_ESTATE_PLACEHOLDER — Manage Estates](../use-cases/UC004_manage_estates.md) |
| **Actor** | PLATFORM_ADMIN |
| **Priority** | MUST HAVE |

## User Story

**As a** PLATFORM_ADMIN **I want to** create a new estate **so that** I can onboard a new property manager on the platform.

## Acceptance Criteria

- AC1: Required fields: `name` (unique, case-insensitive). Optional: `description`.
- AC2: On success: estate created, HTTP 201 returned with `EstateDTO`.
- AC3: Duplicate name → HTTP 409 "An estate with this name already exists."
- AC4: Optional `firstManagerId`: if provided, the referenced user is automatically assigned as MANAGER of the new estate.
- AC5: If `firstManagerId` does not reference an existing user → HTTP 409.
- AC6: Estate immediately visible in the admin estate list.

## Endpoint

`POST /api/v1/admin/estates`

## Request Body

```json
{
  "name": "My Estate",
  "description": "Optional description",
  "firstManagerId": 42
}
```

## Response

HTTP 201 — `EstateDTO`

```json
{
  "id": "a3f8c2d1-7b4e-4f2a-9c1d-8e5f3a2b1c0d",
  "name": "My Estate",
  "description": "Optional description",
  "memberCount": 1,
  "buildingCount": 0,
  "createdAt": "2026-04-12T10:00:00",
  "createdByUsername": "admin"
}
```

## Business Rules

- BR-UC004_ESTATE_PLACEHOLDER-01: Name unique case-insensitively
- BR-UC004_ESTATE_PLACEHOLDER-05: PLATFORM_ADMIN is not added as a member automatically
