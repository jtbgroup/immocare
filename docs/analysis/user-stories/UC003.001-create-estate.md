# UC003.001 — Create Estate

## Overview

| Attribute | Value |
|---|---|
| **ID** | UC003.001 |
| **UC** | [UC003 — Manage Estates](../use-cases/UC-003-manage-estates.md) |
| **Actor** | PLATFORM_ADMIN |
| **Priority** | MUST HAVE |

## User Story

**As a** PLATFORM_ADMIN **I want to** create a new estate and optionally add its initial members **so that** I can onboard a new property manager and their team on the platform in a single operation.

## Acceptance Criteria

- AC1: Required fields: `name` (unique, case-insensitive). Optional: `description`.
- AC2: On success: estate created, HTTP 201 returned with `EstateDTO`.
- AC3: Duplicate name → HTTP 409 "An estate with this name already exists."
- AC4: Optional `members` list: each entry requires a `userId` and a `role` (MANAGER or VIEWER). Multiple members can be added at creation time.
- AC5: If `members` is provided and non-empty, at least one entry must have role `MANAGER` → HTTP 422 if violated.
- AC6: If any `userId` in the `members` list does not reference an existing `app_user` → HTTP 404.
- AC7: Duplicate `userId` values within the `members` list are deduplicated silently (idempotent).
- AC8: Estate and all members are created atomically in the same transaction.
- AC9: Estate immediately visible in the admin estate list.
- AC10: The UI provides an inline member panel in the creation form allowing the user to search users, assign roles, and build the members list before submitting.

## Endpoint

`POST /api/v1/admin/estates`

## Request Body

```json
{
  "name": "My Estate",
  "description": "Optional description",
  "members": [
    { "userId": 42, "role": "MANAGER" },
    { "userId": 17, "role": "VIEWER" }
  ]
}
```

## Response

HTTP 201 — `EstateDTO`

```json
{
  "id": "a3f8c2d1-7b4e-4f2a-9c1d-8e5f3a2b1c0d",
  "name": "My Estate",
  "description": "Optional description",
  "memberCount": 2,
  "buildingCount": 0,
  "createdAt": "2026-04-24T10:00:00",
  "createdByUsername": "admin"
}
```

## Business Rules

- BR-UC003-01: Name unique case-insensitively
- BR-UC003-02: If members list is non-empty, at least one must have role MANAGER
- BR-UC003-05: PLATFORM_ADMIN is not added as a member automatically
- BR-UC003-15: Duplicate userIds within the members list are deduplicated silently
