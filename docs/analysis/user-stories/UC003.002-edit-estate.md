# UC003.002 — Edit Estate

## Overview

| Attribute | Value |
|---|---|
| **ID** | UC003.002 |
| **UC** | [UC003 — Manage Estates](../use-cases/UC-003-manage-estates.md) |
| **Actor** | PLATFORM_ADMIN, MANAGER |
| **Priority** | MUST HAVE |

## User Story

**As a** PLATFORM_ADMIN or MANAGER **I want to** edit an estate's information and manage its members **so that** I can keep the estate accurate and control who has access.

## Acceptance Criteria

- AC1: Editable fields via the general form: `name`, `description`.
- AC2: Name uniqueness enforced excluding the current estate (case-insensitive).
- AC3: Returns HTTP 404 if estate not found.
- AC4: Returns HTTP 409 if new name conflicts with an existing estate.
- AC5: The edit form has two sections: **General Info** (name, description) and **Members** (full member management inline).
- AC6: The Members section provides the same capabilities as the standalone Members page: add member, edit role, remove member — with the same business rule enforcement (last manager, self-operation).
- AC7: MANAGER accesses the edit form from the "Edit Estate" button on the estate dashboard.
- AC8: PLATFORM_ADMIN accesses the edit form from both the admin estate list and the estate dashboard.
- AC9: Both actors use the same `AdminEstateFormComponent`; the route determines which controller endpoint is called.

## Endpoints

| Actor | Method | Endpoint |
|---|---|---|
| PLATFORM_ADMIN | `PUT` | `/api/v1/admin/estates/{id}` |
| MANAGER, PLATFORM_ADMIN | `PUT` | `/api/v1/estates/{id}` |

Both endpoints call the same `EstateService.updateEstate()` method. The estate-scoped route (`/api/v1/estates/{id}`) is the primary endpoint used by the frontend for both actors.

## Request Body

```json
{
  "name": "Updated Estate Name",
  "description": "Updated description"
}
```

## Response

HTTP 200 — `EstateDTO`

## Frontend Routes

| Actor | Route |
|---|---|
| PLATFORM_ADMIN | `/admin/estates/:id/edit` |
| MANAGER | `/estates/:estateId/edit` |

## Business Rules

- BR-UC003-01: Name unique case-insensitively (excluding self)
- BR-UC003-02: Member management in the Members section enforces the last-manager constraint
- BR-UC003-14: MANAGER can edit estate metadata via the estate-scoped PUT endpoint
