# User Story UC013.004: Update General Settings

| Attribute | Value |
|-----------|-------|
| **Story ID** | UC013.004 |
| **Epic** | Platform Administration |
| **Related UC** | UC013 |
| **Priority** | MUST HAVE |
| **Story Points** | 3 |

**As an** ADMIN **I want to** update the application settings **so that** I can control alert thresholds and asset classification behaviour.

## Acceptance Criteria

### General Settings

**AC1:** General Settings section shows all configurable parameters with current values: "Boiler service expiry warning threshold" (e.g. "3 months") and an "Edit" button per parameter.

**AC2:** Click "Edit" → inline form with the current value pre-filled.

**AC3:** Save valid value → config updated, new value shown immediately.

**AC4:** Value ≤ 0 or not an integer → error "Value must be a positive integer".

**AC5:** Cancel → original value restored, no change saved.

**AC6:** Updated at timestamp and updated by username shown next to each setting.

### Asset Type Mappings

**AC7:** Asset Type Mappings section shows one row per asset type (BOILER, FIRE_EXTINGUISHER, METER): asset type label, currently mapped subcategory (shown as "Category / Subcategory") or "Not mapped".

**AC8:** Click "Edit" on a row → inline form with: category dropdown + subcategory dropdown (filtered to selected category, direction EXPENSE or BOTH only) + "Clear mapping" button.

**AC9:** Select a subcategory → save → mapping updated immediately. "Not mapped" row becomes "Category / Subcategory".

**AC10:** Click "Clear mapping" → mapping removed, row shows "Not mapped" again.

**AC11:** Subcategory with direction INCOME cannot be selected → error "Asset type mappings must use a subcategory with direction EXPENSE or BOTH".

**AC12:** When an asset link of a mapped type is added to a transaction, the mapped subcategory is automatically pre-filled on the transaction. The admin can override freely.

**AC13:** Cancel → original mapping restored, no change saved.

**Endpoints:**
- `PUT /api/v1/config/settings/{key}` — HTTP 200 (general settings)
- `GET /api/v1/config/asset-type-mappings` — HTTP 200
- `PUT /api/v1/config/asset-type-mappings/{assetType}` — HTTP 200
