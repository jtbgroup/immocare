# User Story UC015.009: Manage Tag Catalog (Categories & Subcategories)

| Attribute | Value |
|-----------|-------|
| **Story ID** | UC015.009 |
| **Epic** | Financial Management |
| **Related UC** | UC015 |
| **Priority** | MUST HAVE |
| **Story Points** | 3 |

**As an** ADMIN **I want to** manage the two-level classification catalog (categories and subcategories) **so that** I can define and evolve the taxonomy used for financial analysis.

## Acceptance Criteria

**AC1:** "Settings" tab on the Transactions page contains two sub-sections: "Categories & Subcategories" and "Bank Accounts" (UC015.010).

**AC2:** Categories section shows all categories in a table ordered alphabetically: name, description, number of subcategories. "Add Category" button opens an inline form row at the top.

**AC3:** Add Category form fields: name* (required, unique case-insensitive), description (optional). Save → category created, appears in list. Duplicate name → error "A category with this name already exists."

**AC4:** Each category row has an "Edit" button (inline edit of name and description) and a "Delete" button.

**AC5:** Delete category blocked if it has any subcategories → error "This category contains [N] subcategory/subcategories. Delete them first." No cascade delete.

**AC6:** Subcategories section below categories (or as an expandable sub-panel per category) shows all subcategories: name, parent category, direction compatibility badge (INCOME / EXPENSE / BOTH), usage count (number of transactions using it), description.

**AC7:** "Add Subcategory" form fields: name* (required, unique within category, case-insensitive), category* (required dropdown), direction* (INCOME / EXPENSE / BOTH), description (optional).

**AC8:** Duplicate subcategory name within the same category → error "A subcategory with this name already exists in this category."

**AC9:** Edit subcategory: all fields editable. Changing `direction` blocked if existing transactions use this subcategory with an incompatible direction → error "Cannot change direction: [N] transaction(s) with direction [X] use this subcategory."

**AC10:** Delete subcategory blocked if `usage_count > 0` → error "This subcategory is used on [N] transaction(s) and cannot be deleted."

**Endpoints:**
- `GET /api/v1/tag-categories` — list all categories with subcategory count.
- `POST /api/v1/tag-categories` — create category.
- `PUT /api/v1/tag-categories/{id}` — update category.
- `DELETE /api/v1/tag-categories/{id}` — delete category.
- `GET /api/v1/tag-subcategories` — list all subcategories (optional filter: `?categoryId=`).
- `POST /api/v1/tag-subcategories` — create subcategory.
- `PUT /api/v1/tag-subcategories/{id}` — update subcategory.
- `DELETE /api/v1/tag-subcategories/{id}` — delete subcategory.
