# User Story UC015.007: Import Transactions (CSV / PDF)

| Attribute | Value |
|-----------|-------|
| **Story ID** | UC015.007 |
| **Epic** | Financial Management |
| **Related UC** | UC015, UC016 |
| **Priority** | MUST HAVE |
| **Story Points** | 8 |

**As an** ADMIN **I want to** import a bank statement file (CSV or PDF) **so that** I can bulk-load transactions without manual entry and benefit from automatic classification and lease suggestions.

## Acceptance Criteria

### Step 1 — Form

**AC1:** "Import" tab shows a two-field form: Parser selector + file upload zone (drag-and-drop or browse).

**AC2:** Parser selector lists all active parsers (`GET /api/v1/import-parsers`), ordered by label. Each entry shows label, format badge (CSV / PDF), bank hint.

**AC3:** File upload zone accepts only the extension matching the selected parser format. Mismatched file → error "This parser expects a {FORMAT} file. Got: .{ext}." Max size 10 MB.

**AC4:** "Preview" button enabled only when both parser and file are selected.

### Step 2 — Preview

**AC5:** Clicking "Preview" calls `POST /api/v1/transactions/preview` (multipart: `file` + `parserCode`). Preview table appears with columns: #, Date, Amount, Direction badge, Description, Counterparty name, Counterparty account, Status (OK / DUPLICATE / ERROR).

**AC6:** Duplicate rows (fingerprint already in DB) shown with a "Duplicate" badge — greyed out, unselectable, not imported.

**AC7:** Parse error rows shown with a red "Error" badge and inline error message. Unselectable.

**AC8:** Valid rows shown with a checkbox (selected by default). Admin can deselect rows to exclude them from import.

**AC9:** Subcategory suggestion shown per row (with confidence score) when `suggestionConfidence ≥ threshold`. Lease suggestion shown when a counterparty IBAN matches a known person — displays person name, unit number, building name.

**AC10:** Clicking a valid row opens a side detail panel (enrichment panel) with fields: Direction override, Bank account, Category + Subcategory (filtered by direction), Building, Housing Unit (cascading), Lease (all statuses: DRAFT / ACTIVE / FINISHED / CANCELLED — to support historical imports). "Accept suggestion" button pre-fills building + unit + lease from the lease suggestion. Changes saved locally in the preview state.

**AC11:** Summary bar shows: Total rows / Valid / Duplicates / Errors / Selected.

### Step 3 — Import

**AC12:** "Import selected rows" button calls `POST /api/v1/transactions/import` (multipart: `file`, `parserCode`, `bankAccountId`, `enrichments` JSON array, `selectedFingerprints` JSON array).

**AC13:** Rows with an enrichment entry (matched by fingerprint) saved as CONFIRMED. Rows without enrichment saved as DRAFT.

**AC14:** Result banner: "Imported: X | Duplicates skipped: Y." "Review" button navigates to `/transactions/import/{batchId}` (UC015.008).

**AC15:** If all rows are duplicates or errors → no `import_batch` created → message "No new transactions to import."

## Endpoints

- `GET /api/v1/import-parsers` — active parser list
- `POST /api/v1/transactions/preview` — multipart: `file`, `parserCode` → `List<ImportPreviewRowDTO>`
- `POST /api/v1/transactions/import` — multipart: `file`, `parserCode`, `bankAccountId`, `enrichments`, `selectedFingerprints` → `ImportBatchResultDTO`
