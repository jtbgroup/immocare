# UC003.013 — Enforce Estate-scoped Data Access

## Overview

| Attribute | Value |
|---|---|
| **ID** | UC003.013 |
| **UC** | [UC004_ESTATE_PLACEHOLDER — Manage Estates](../use-cases/UC004_manage_estates.md) |
| **Actor** | System (cross-cutting) |
| **Priority** | MUST HAVE |

## User Story

**As the** system **I want to** ensure that all data queries are scoped to the active estate **so that** no data leaks between estates.

## Acceptance Criteria

- AC1: All repository queries for estate-scoped entities include an `estate_id` filter.
- AC2: A user attempting to access a resource belonging to a different estate receives HTTP 403.
- AC3: PLATFORM_ADMIN can read and write data from any estate by including the `estateId` in the URL path.
- AC4: The `estateId` is always explicit in the URL path — never inferred from session or implicit state.
- AC5: A Spring `EstateSecurityService` (`@Service("security")`) is used in `@PreAuthorize` expressions to verify membership and role per request.

## Implementation Notes

No dedicated API endpoint — this is a cross-cutting concern implemented via:

**Backend:**
- `EstateSecurityService` (`@Service("security")`) exposes methods used in `@PreAuthorize`:
  - `isPlatformAdmin()`
  - `isManagerOf(UUID estateId)`
  - `isMemberOf(UUID estateId)` — MANAGER or VIEWER
- Each controller method annotated with the appropriate expression
- Each service method receives `estateId` as an explicit parameter — no ThreadLocal

**Frontend:**
- `ActiveEstateService` stores active estate as an Angular signal
- All HTTP service methods include `estateId` in the URL path
- `EstateGuard` checks membership before activating estate-scoped routes
- Action buttons (Create, Edit, Delete) hidden when `ActiveEstateService.isViewer()` is true

## Business Rules

- BR-UC004_ESTATE_PLACEHOLDER-06: `estateId` always explicit in URL — never server-side session
- BR-UC004_ESTATE_PLACEHOLDER-07: Estate-scoped queries always filtered by `estate_id`
- BR-UC004_ESTATE_PLACEHOLDER-08: VIEWER cannot mutate any resource
